# Magister Marcus
### *The New World Made Ready* — A Family History & Civics Curriculum

A Charlotte Mason–style, literature-based American history and civics curriculum for upper-elementary families (grades 3–6), built on the Hillsdale College K–12 American History content framework. Designed for home use and structured for resale.

---

## What This Is

A complete two-year family study curriculum covering American history from 1492 to 1877, organized into four units:

| Unit | Period | Dates |
|------|--------|-------|
| **Unit 1** | The British Colonies of North America | 1492–1763 |
| **Unit 2** | The American Founding | 1763–1789 |
| **Unit 3** | The Early Republic | 1789–1848 |
| **Unit 4** | The American Civil War Era | 1848–1877 |

**Pace:** 4 lessons per week · ~9 weeks per unit · 36 weeks per school year · 2 years total

**Pedagogy:** Charlotte Mason — narration (oral and written) as primary assessment; living books; copywork and dictation from primary sources; notebooking pages with student illustration and narration; no textbook-style comprehension packets.

**Faith posture:** Quietly Christian — Providence framing where it arises naturally in the historical record; no explicit devotional content; no sectarian material.

---

## Who This Is For

- Homeschool families with children in grades 3–6
- Families studying together across multiple ages (differentiated by Form IA / Form IIA)
- Parents who want a rigorous, story-driven, low-cost American history curriculum
- Those already using or familiar with the Hillsdale K–12 American History framework who want richer student materials

---

## Two Student Forms

Every lesson, assessment, and notebooking page exists in two differentiated forms:

- **Form IA** — Grades 3–4, developing writers. Oral narration, sentence-level written responses, matching and fill-in-the-blank assessments, simpler copywork.
- **Form IIA** — Grades 5–6, stronger writers. Written paragraph narration, short essay assessments, dictation passages, fuller notebooking responses.

Both forms cover identical content. Families may assign forms individually or use one form for all children.

## How to Use the Lesson Plans

Each lesson plan follows a four-day CM weekly rhythm:

| Day | Focus |
|-----|-------|
| **Day 1** | Geography, context, or map work |
| **Day 2** | Core narrative lesson — the "story day" |
| **Day 3** | Primary source, biography, or living book chapter |
| **Day 4** | Narration (oral or written), copywork, or notebooking |

Each lesson plan includes:
- **Teacher narrative notes** — what to tell and how to frame it
- **Key to the Lesson** — the one idea that must land
- **Story for the Heart** — the narrative moment to dwell in
- **Questions for the Mind** — discussion questions at both form levels
- **CM narration prompts** (Form IA: oral; Form IIA: written paragraph)
- **Copywork/dictation passage** for the week
- **Civics connections** flagged with ⚖️

---

## Assessment Philosophy

This curriculum follows Charlotte Mason's conviction that **narration is the primary assessment**. Students who can tell back a story in their own words have understood it.

Formal assessments exist for families who want them and for the curriculum's resale value:

- **Formative quizzes** (3 per unit) — matching, fill-in-the-blank, 2–4 sentence responses
- **Two unit tests** — timeline matching, vocabulary matching, multiple choice, short written response
- **Unit writing assignment** — one paragraph (Form IA) or 2–3 paragraphs (Form IIA) synthesizing the unit

All assessments follow the format established in the Hillsdale 3–5 grade materials, adapted for CM context.

# Magister Marcus — Via Latina

A classical Latin tutor for elementary students built with React + Cloudflare Pages.  
Teaches reading, translation, and pronunciation of authentic historical texts.

## Features

- **4 student profiles** — isolated history, level, and text per student
- **5 curriculum levels** — Gradus I (beginner) through Gradus V (Virgil, Horace, Ovid)
- **12 historical texts** — Caesar, Cicero, Virgil, Phaedrus, Vulgate, inscriptions, and more
- **Text-to-Speech** — every Latin passage has a ▶ RECITA button with classical pronunciation
- **Speech recognition** — students speak answers aloud (Chrome/Edge)
- **Pronunciation Drill** — graded listen-and-repeat mode with scoring
- **Speed control** — adjustable TTS rate from Lentissime to Celerrime

## Deployment

### 1. Push to GitHub

```bash
# In this directory
git init
git add .
git commit -m "Initial commit — Magister Marcus"

# Create a new repo at github.com, then:
git remote add origin https://github.com/YOUR_USERNAME/magister-marcus.git
git branch -M main
git push -u origin main
```

### 2. Connect to Cloudflare Pages

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com) → **Workers & Pages** → **Create**
2. Choose **Pages** → **Connect to Git**
3. Select your `magister-marcus` repository
4. Set build configuration:
   - **Framework preset:** `Vite`
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
5. Click **Save and Deploy**

### 3. Add the Anthropic API key as a secret

> This is the critical step — the key lives only in Cloudflare, never in your code.

1. In Cloudflare Dashboard → your Pages project → **Settings** → **Environment variables**
2. Click **Add variable** under **Production**
3. Set:
   - **Variable name:** `ANTHROPIC_API_KEY`
   - **Value:** your key from [console.anthropic.com](https://console.anthropic.com/settings/keys)
   - Check **Encrypt** ← important
4. Click **Save**
5. Go to **Deployments** → **Retry deployment** (so it picks up the new secret)

Your site will be live at `https://magister-marcus.pages.dev` (or your custom domain).

## Local Development

You need Node.js 18+ and a Cloudflare account.

```bash
# Install dependencies
npm install

# Install wrangler (Cloudflare CLI) globally if you haven't
npm install -g wrangler

# Copy the example secrets file
cp .dev.vars.example .dev.vars
# Edit .dev.vars and add your real ANTHROPIC_API_KEY

# Terminal 1 — run the Pages Functions (API proxy) on port 8788
npx wrangler pages dev dist --port 8788

# Terminal 2 — run Vite dev server (proxies /api/* to wrangler)
npm run dev
```

Then open http://localhost:5173

> **Note:** Wrangler needs the `dist` folder to exist for local Pages Functions dev.  
> Run `npm run build` once first if you haven't.

## Adding a New Subject

1. Create `src/pages/YourSubject.jsx`
2. Add a route in `src/main.jsx`:
   ```jsx
   <Route path="/your-subject" element={<YourSubject />} />
   ```
3. Add a card in `src/pages/Home.jsx` in the `TOOLS` array:
   ```js
   { id:'your-subject', icon:'📚', title:'Your Tool', subtitle:'...', href:'/your-subject', status:'active', tag:'Active' }
   ```
4. Push to GitHub — Cloudflare deploys automatically.
